Kết nối Backend và Frontend

   

Phát triển Web API

   

Trong phát triển ứng dụng web hiện đại, Frontend (giao diện người dùng) và Backend (logic và dữ liệu) thường hoạt động độc lập và giao tiếp với nhau thông qua một giao diện được chuẩn hóa: **API** (Application Programming Interface).

1. API: Cầu nối giao tiếp

Hầu hết các kết nối giữa hai bên được thực hiện bằng cách sử dụng **RESTful API** (hoặc GraphQL), dựa trên các yêu cầu **HTTP**:

  1. Yêu cầu (Request): Frontend (sử dụng JavaScript) gửi một yêu cầu HTTP (GET, POST, PUT, DELETE) đến một **Endpoint** (URL) cụ thể trên Server.
  2. Xử lý (Processing): Backend nhận yêu cầu, xử lý logic nghiệp vụ, và tương tác với cơ sở dữ liệu (Database).
  3. Phản hồi (Response): Backend trả về dữ liệu (thường ở định dạng **JSON**) và một **Mã trạng thái HTTP** (ví dụ: 200 OK, 401 Unauthorized) cho Frontend.

2. Ví dụ về luồng dữ liệu

Giả sử người dùng muốn lấy danh sách sản phẩm:

  • Frontend: Gửi GET /api/products.
  • Backend: Truy vấn Database, nhận về danh sách sản phẩm.
  • Response (JSON): [{"id": 1, "name": "Laptop"}, {"id": 2, "name": "Mouse"}], kèm theo Status 200.
  • Frontend: Nhận JSON, dùng JavaScript để tạo các thẻ HTML và hiển thị danh sách sản phẩm lên trang web.

Việc tách biệt này giúp cả hai đội ngũ Frontend và Backend có thể phát triển độc lập và linh hoạt hơn.
