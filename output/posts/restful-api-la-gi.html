RESTful API là gì?

   

API Kiến trúc

   

REST (Representational State Transfer) là một kiểu kiến trúc phần mềm, được Roy Fielding giới thiệu năm 2000, sử dụng giao thức HTTP làm giao diện trung chuyển. Một **RESTful API** là một Web API tuân thủ các nguyên tắc thiết kế REST.

1. Các nguyên tắc cốt lõi

  • Client-Server: Tách biệt giao diện người dùng và việc xử lý dữ liệu.
  • Stateless (Không trạng thái): Server không lưu trữ bất kỳ dữ liệu phiên (session) nào của Client giữa các yêu cầu. Mỗi Request phải độc lập.
  • Sử dụng Tài nguyên (Resource): Mọi thứ đều được coi là tài nguyên, được định danh bằng **URL** (ví dụ: /users, /products/1).

2. Ánh xạ HTTP Methods (CRUD)

RESTful API sử dụng các phương thức HTTP tiêu chuẩn để thao tác với tài nguyên:

MethodMục đíchVí dụ URL
**GET**Đọc/Truy xuất tài nguyên/users/1
**POST**Tạo mới một tài nguyên/users
**PUT**Cập nhật toàn bộ tài nguyên/users/1
**DELETE**Xóa một tài nguyên/users/1

Việc sử dụng các phương thức HTTP một cách nhất quán giúp API trở nên dễ hiểu, dễ sử dụng và dễ mở rộng.
